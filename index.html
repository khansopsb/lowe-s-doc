Forecast/History sheet
Fill rate
Lowe's fill rate% ratio
What percentage of the units Lowe's ordered from PrimeSource were actually received by Lowe's. A fill rate of 95% means 5% of ordered units did not arrive. Uses Lowe's receiving data (LOWES_FR_RECEIVED_QTY and LOWES_FR_ORDERED_QTY).
Lowe's fill rate (current)% ratio
Same ratio as Lowe's Fill Rate but restricted to only the most recent month that has actual order data. The workbook finds that month automatically using a FIXED level-of-detail calculation — you never need to set a date filter manually. This is the number shown in the Current Fill Rate Opportunities view.
Lowe's fill rate misses (current)unit gap
The raw number of units ordered but not received in the current month. Where fill rate tells you the percentage gap, this tells you the size of the gap in actual units — useful for prioritizing which products to chase.
PS fill rate% ratio
PrimeSource's fill rate measured using PS's own fulfillment records (FR_INVOICED_QTY and FR_ORDER_QTY), as opposed to Lowe's receiving data. Can differ from Lowe's Fill Rate due to timing differences, transit lags, or how each side records receipts.
PS fill rate CY / LY / % diff (bps)year-over-year
Year-over-year split of PS Fill Rate. CY = current calendar year months; LY = the same months one year prior. The % Diff is expressed in basis points — 100 bps equals 1 percentage point — so small changes are readable. Going from 92% to 94% fill rate = +200 bps.
PS fill rate misses (current)unit gap
Same concept as Lowe's Fill Rate Misses but using PrimeSource's own order and invoiced quantities for the most recent available month.
Sales & forecast variance
Lowe's var (Lowe's fcst vs invoiced sales)unit variance
How many units Lowe's forecast missed PS Invoiced Sales by. Positive = Lowe's over-forecasted (expected more than PS shipped). Negative = under-forecasted. Used to judge the accuracy of Lowe's own demand signal.
PS var (PS fcst vs PS invoiced sales)unit variance
How far PrimeSource's own forecast missed actual PS invoiced shipments. Positive = PS over-forecasted.
POS var (Lowe's POS vs PS invoiced sales)unit variance
Difference between what consumers bought at Lowe's (POS) and what PS actually shipped to Lowe's (invoiced). A persistently positive number means Lowe's is selling through product faster than PS is replenishing it — a potential stockout risk.
Lowe's % / PS % / POS %% variance
Each of the three variances above expressed as a percentage of PS Invoiced Sales. Lets you compare forecast accuracy across products of very different sizes on a level playing field.
Sales and forecast (blended line)blended
A single continuous series used in the PS Invoiced Sales/Forecast chart. For months that already have actual data it shows PS Invoiced units; for future months it seamlessly switches to PS Forecast. The cutover point is the latest month with invoiced data — determined automatically, no manual date needed.
Year-over-year (YoY)
PS invoiced CY / LY / % diffyear-over-year
PS Invoiced units split into current year and prior year for direct comparison. % Diff shows whether shipments are running above or below last year.
PS forecast CY / LY / diffyear-over-year
Same structure as PS Invoiced YoY but for the PS Forecast field. Shows whether forecast levels have changed year over year, independent of actual performance.
Lowe's invoiced % differenceyear-over-year
Year-over-year change in Lowe's retail sales dollars (not units). Uses the Sales $ TY field.
Store & inventory metrics
Store countconfigurable
The denominator used in all "per store" metrics. Controlled by the Store Count to Use parameter. "Total Stores" averages the full Lowe's chain count (ACTIVE_COUNT_SYSTEM) regardless of which SKUs are stocked. "Stores where SKU is Active" sums only stores that carry the specific product being viewed — tighter and product-specific.
Lowe's inv/strper-store ratio
Average Lowe's on-hand inventory per store. Normalizes inventory for store count so you can fairly compare across time periods or product groups. A declining trend here alongside stable POS can signal an approaching stockout.
Lowe's inv/str CY / LY / % diffyear-over-year
Year-over-year comparison of inventory per store. Useful for seeing whether in-store availability has improved or tightened versus the same time last year.
Lowe's sales/str/SKUper-store ratio
Consumer sell-through (POS units) per store — the velocity metric. Shows how fast each product moves at store level. Use alongside Inv/Str to assess weeks of cover: high sales/str + low inv/str = stock running out fast.
Lowe's sales/str/SKU CY / LY / diffyear-over-year
Year-over-year comparison of per-store sell-through. The diff is an absolute unit difference (CY minus LY), not a percentage, to avoid distortion from small denominators.
Active stores (current)count
Number of Lowe's stores stocking this SKU in the most recent data month. Uses the same auto-detected "latest month" logic as the fill rate current fields — no manual date filter needed.
Active stores LYyear-over-year
Active store count for the same calendar month one year ago. Compare against Active Stores (Curr) to see if distribution has grown or contracted.
On order (current)units
Units Lowe's has on open purchase orders from PrimeSource as of the most recent available month. Represents the inbound pipeline — ordered but not yet received in stores.
History vs forecast (label)label field
A text label automatically applied to each month. Months before the current calendar month are tagged "History"; the current month and future months are tagged "Forecast". Used to color-code bars in the Forecast/History dashboard so the boundary between actuals and projections is always visible.
Transaction & volume
DATA_MTHtext — YYYY-MM
The period month stored as text in YYYY-MM format (e.g. "2024-03"). All year-over-year logic reads the first 4 characters as the year and the last 2 as the month. The calculated field Month converts this to a proper date for chart axis labels.
INVOICED / PS invoicedunits
Units that PrimeSource actually invoiced (shipped) to Lowe's in the given month. This is the primary actuals field — every "PS Invoiced Sales" reference in the dashboard points here. It is the benchmark all forecasts are compared against.
INVOICED_SALESdollars
Dollar value of PrimeSource's invoiced shipments. Used in the PS Invoiced Sales CY/LY year-over-year comparison on the YoY sheet.
MONTHLY_FORECAST / Lowe's forecastunits — forecast
Lowe's unit forecast for each month — what Lowe's system projects it will need. This is the Lowe's demand signal used in all Lowe's variance calculations. It comes from Lowe's, not PrimeSource.
PS_FORECAST / PS forecastunits — forecast
PrimeSource's own unit forecast. Compared against INVOICED to measure how accurately PS predicted its own shipment volumes. Also shown as the future portion of the blended Sales and Forecast line.
Lowe's sales units (Sales Units TY)units — consumer POS
Units sold by Lowe's to consumers at the register — point-of-sale data. "TY" = This Year. This measures consumer demand, which is distinct from what PS shipped to Lowe's. Comparing POS to INVOICED shows the replenishment gap.
Lowe's sales $ (Sales $ TY)dollars — consumer POS
Dollar value of Lowe's consumer POS sales in the current year. Used in the Lowe's Invoiced % Difference YoY metric.
LOWES_FR_ORDERED_QTYunits
Units that Lowe's placed on fill-rate orders with PrimeSource. This is the denominator of Lowe's Fill Rate — what was requested.
LOWES_FR_RECEIVED_QTYunits
Units that Lowe's physically received against fill-rate orders. This is the numerator of Lowe's Fill Rate — what actually arrived.
FR_ORDER_QTYunits
Units ordered as recorded in PrimeSource's system. The denominator of PS Fill Rate. May differ slightly from LOWES_FR_ORDERED_QTY because each system records orders independently — timing and system differences can create small gaps.
FR_INVOICED_QTYunits
Units invoiced as recorded in PrimeSource's system. The numerator of PS Fill Rate.
Lowe's on hand (units) / available inventoryunits
Units currently on Lowe's store shelves and backrooms. Used as the numerator in the Inv/Str metric and plotted in the Lowe's Inventory trend chart.
Lowe's on order (units) / inventory on direct orderunits
Units Lowe's has on open purchase orders — ordered from PrimeSource but not yet received. Represents what's in the inbound pipeline. Shown in the On Order trend chart and the current-period metrics.
LOWES_ACTIVE_STOREcount
Number of Lowe's stores where a specific SKU is listed as active/stocked. Used as the store count denominator when "Stores where SKU is Active" is selected. Varies by product — an SKU in only 400 stores shows 400 here even if Lowe's has many more total locations.
ACTIVE_COUNT_SYSTEMcount
Total number of active Lowe's stores chain-wide, regardless of which SKUs are stocked there. Used when "Total Stores" is the selected parameter. Averaged (AVG) rather than summed because the same total store count repeats for every row in a given month.
PS_SKU / Paste SKU listID
PrimeSource's internal product identifier. The Paste SKU List filter uses this field — paste a list of PS SKU numbers to scope all views to specific products. Leave blank to see all SKUs.
LOWES_SKUID
Lowe's own item number for the same product. When aggregating by Material ID in the variance table, both identifiers are shown together (PS SKU — Lowe's SKU) so you can look up either side.
FLC major / FLC minorproduct hierarchy
Two-level product category hierarchy used throughout the dashboard. FLC Major is the broader grouping (e.g. a product family); FLC Minor is a subdivision within it. The default aggregation in most views rolls up by FLC Major. Most filter panels let you pick one or both levels to drill down.
PS_DC / Plant IDlocation
The PrimeSource distribution center or manufacturing plant that fulfills the product. Available as a dimension in the Dimensions parameters and Aggregation Filter so you can break down any metric by which DC is shipping.
Hub / Districtsgeography
Geographic groupings of Lowe's store locations. Hub is the broader regional level; Districts is more granular. Available as filters on most sheets — useful for isolating performance in a specific part of the country.
ITEM_STATUStext
Flags whether an item is active or inactive. The calculated field item s converts this to "Show" or "Hide" — null values, blank strings, and the literal text "Null" all map to "Hide", so inactive or missing-status items are automatically excluded from views without any manual filter work.
PS inventoryunits
PrimeSource's own on-hand inventory at its distribution centers — distinct from inventory sitting in Lowe's stores. Plotted in the PS Inventory trend chart on the metrics charts dashboard.
OPEN POS / Sales orders / PO RCVDunits
Supply chain pipeline fields shown in the main Lowe's summary sheet. Open POS = Lowe's open purchase orders to PrimeSource. Sales Orders = PrimeSource's sales order backlog. PO RCVD = purchase orders Lowe's has confirmed received. Together they show where product is in the supply chain at any moment.

How "current period" auto-detection works
DATA_THRU fields — how the workbook finds the latest month automaticallytechnical
Many metrics show a "current" or "Curr" value without you having to set a date filter. The workbook finds the most recent month with actual data for each metric using a FIXED level-of-detail (LOD) expression. FIXED means the calculation runs across the entire dataset and ignores any filters on the view — so the "latest month" is always global. Each metric has its own DATA_THRU because different data types (fill rate, store count, inventory, on-order) can have different data lag times and may not all be current to the same month.
